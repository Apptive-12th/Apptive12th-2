아키텍처 패턴
	아키텍처 내에서 계층을 분리 함으로 써 각 단위의 이해, 수정, 이동, 공유, 테스트 등에 있어서
	상당한 장점이 있기 때문에 현대 소프트웨어 에서는 아키텍처 패턴이 매우 중요하다.

MVC 모델 ( Model View Controller )
  Model : 모든 데이터, 상태 및 어플리케이션 로직 이 들어있다.
  View : 모델을 표현하는 형태 로써 대표적으로 사용자 인터페이스 가 있다.
  Controller : 사용자로부터 입력을 받고 그에 따른 행동 지시를 모델에게 내린다.


예를 들어 메모 어플에서
View 는 현재 화면에 보이는 메모입력화면 그 자체이고 여기서 메모 입력 후
저장 버튼을 누르면 Controller 가 저장 버튼이 눌려짐을 확인 하고 Model 에게 
해당 메모 data 를 DB 에 저장하라고 지시 한다. Model 에서 구현되어 있는 logic 을 통해 메모 저장 후
View 에게 저장이 되었다고 알려준다.
